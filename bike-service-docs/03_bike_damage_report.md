# 车辆损坏上报规范

## 问题类型
- **问题名**: `BikeDamageReport`
- **问题级别**: 中优先级
- **触发条件**: 用户发现单车存在损坏或安全隐患

## 问题描述
用户在使用单车前或使用过程中发现车辆存在损坏，常见情况：
- 车座松动或损坏
- 刹车失灵或刹车片磨损
- 车轮变形、爆胎
- 车把手松动
- 脚踏板损坏
- 车身严重锈蚀
- 车锁损坏（无法上锁）
- 显示屏/仪表盘故障（电动单车）
- 电池续航异常（电动单车）

## 安全提示
**重要**: 如用户反映刹车失灵、车轮变形等安全隐患，必须第一时间提醒用户**立即停止使用**，确保人身安全。

## 排查步骤

### 步骤1: 确认损坏类型
引导用户描述损坏情况，区分：
- **安全隐患类**（刹车、车轮、车把）：立即停用，优先处理
- **舒适性问题类**（车座、脚踏）：建议换车，记录上报
- **功能性问题类**（车锁、显示屏）：记录上报，安排维修

### 步骤2: 记录车辆信息
**必须收集**:
- 单车编号（车身二维码下方）
- 损坏描述（尽量详细）
- 发现位置（GPS 或地址描述）
- 发现时间
- 用户是否拍照留存

### 步骤3: 判断责任归属
**查询内容**:
- 该车辆最近一次骑行记录（上一位用户）
- 该车辆历史维修记录
- 用户本次是否已开始骑行

**责任判断**:
- 用户开锁前发现损坏 → 非用户责任，感谢用户上报
- 用户骑行中发现损坏 → 需核实是否骑行前已存在
- 用户还车后被下一位用户投诉 → 需调查损坏时间点

## 常见损坏类型与处理方案

### 类型1: 刹车问题（高优先级）
**安全风险**: 高
**处理方案**:
1. 立即提醒用户停止使用，确保安全
2. 在系统中将该车辆标记为"故障停用"
3. 自动派单给最近的维修人员（响应时间 ≤ 2小时）
4. 感谢用户上报，补偿 **1张30分钟免费骑行券**
5. 引导用户使用附近其他单车

### 类型2: 车轮/爆胎问题（高优先级）
**安全风险**: 高
**处理方案**:
1. 立即提醒用户停止使用
2. 标记车辆为"故障停用"
3. 派单维修（响应时间 ≤ 2小时）
4. 补偿 **1张30分钟免费骑行券**

### 类型3: 车座/车把松动（中优先级）
**安全风险**: 中
**处理方案**:
1. 建议用户换一辆单车
2. 记录故障车辆，派单维修（响应时间 ≤ 24小时）
3. 补偿 **1张5分钟免费骑行券**

### 类型4: 车锁故障（中优先级）
**安全风险**: 低（但影响使用）
**处理方案**:
1. 如无法上锁：引导用户联系客服手动结束订单
2. 标记车辆故障，派单维修
3. 如用户因此产生额外费用，退还相关费用

### 类型5: 电动单车电池续航异常
**特征**: 显示电量充足但实际续航很短
**处理方案**:
1. 记录车辆编号和用户反馈
2. 派单给电动车专项维修团队
3. 补偿 **1张骑行券**

## 用户骑行中发生损坏的处理

### 骑行中车辆损坏导致用户受伤
**紧急处理**:
1. 第一时间询问用户是否受伤，如受伤立即建议就医
2. 告知用户保留现场照片和就医记录
3. 立即上报给主管，启动应急处理流程
4. 不得擅自承诺赔偿金额，需等待主管介入

**后续处理**:
- 联系保险部门，启动意外险理赔流程
- 配合用户提供骑行记录作为证明材料

### 骑行中车辆损坏未造成受伤
1. 手动结束订单，费用按实际骑行时间计算
2. 如损坏影响骑行体验，可酌情减免本次费用
3. 补偿 **1张30分钟免费骑行券**

## 维修派单标准

| 损坏类型 | 响应时间 | 处理优先级 |
|---------|---------|-----------|
| 刹车失灵 | ≤ 2小时 | P0 紧急 |
| 车轮变形/爆胎 | ≤ 2小时 | P0 紧急 |
| 车把松动 | ≤ 4小时 | P1 高 |
| 车座损坏 | ≤ 24小时 | P2 中 |
| 车锁故障 | ≤ 4小时 | P1 高 |
| 外观损坏 | ≤ 72小时 | P3 低 |

## 补偿标准
- 上报安全隐患（刹车/车轮）：**1张30分钟免费骑行券**（感谢上报）
- 上报一般损坏：**1张5分钟免费骑行券**
- 骑行中遭遇损坏影响体验：**1张30分钟免费骑行券** + 酌情减免本次费用

## 验证步骤
1. 确认维修工单已创建并派单
2. 确认车辆已在系统中标记为停用
3. 确认补偿已发放到用户账户
4. 24小时后跟进维修完成情况
