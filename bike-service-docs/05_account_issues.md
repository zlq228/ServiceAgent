# 账户问题处理规范

## 问题类型
- **问题名**: `AccountIssue`
- **问题级别**: 高优先级
- **触发条件**: 用户账户出现异常，无法正常使用服务

## 问题描述
用户账户出现异常，常见表现：
- 无法登录账户
- 账户被封禁/限制使用
- 手机号更换后无法找回账户
- 账户余额异常（莫名减少）
- 骑行记录丢失
- 优惠券/骑行卡消失
- 实名认证失败
- 账户被盗用

## 账户状态说明

| 账户状态 | 说明 | 影响 |
|---------|------|------|
| 正常 | 可正常使用所有功能 | 无 |
| 限制使用 | 有未处理违规记录 | 无法开锁 |
| 临时封禁 | 违规次数过多 | 封禁期内无法使用 |
| 永久封禁 | 严重违规行为 | 永久无法使用 |
| 注销中 | 用户申请注销 | 7天冷静期内可撤销 |

## 排查步骤

### 步骤1: 验证用户身份
**必须核实**:
- 用户注册手机号
- 实名认证姓名（部分场景需要）
- 最近一次骑行时间和地点（辅助验证）

### 步骤2: 查询账户状态
**查询内容**:
- 账户当前状态
- 封禁/限制原因
- 封禁开始时间和预计解除时间
- 违规记录列表

### 步骤3: 查询账户变动记录
**查询内容**:
- 余额变动流水
- 优惠券使用/过期记录
- 登录记录（IP、设备、时间）

## 常见原因与处理方案

### 原因1: 账户被封禁
**特征**: 用户反映无法开锁，提示"账户已被限制"
**处理方案**:
1. 查询封禁原因，告知用户
2. 常见封禁原因及解除方式：

| 封禁原因 | 解除条件 |
|---------|---------|
| 违规停车罚款未缴 | 缴清罚款后自动解除 |
| 恶意损坏车辆 | 缴纳赔偿费用后解除 |
| 违规次数超限（3次） | 完成安全教育课程后解除 |
| 账户欠费 | 补缴欠费后解除 |
| 投诉核实违规 | 视情节轻重，1-30天封禁 |

3. 引导用户按对应方式解除封禁
4. 如用户对封禁有异议，记录申诉信息，上报审核

### 原因2: 无法登录
**特征**: 用户反映验证码收不到或登录失败
**处理方案**:
1. 确认用户手机号是否正确
2. 验证码收不到：
   - 检查手机是否屏蔽了短信
   - 建议用户尝试语音验证码
   - 如仍失败，核实手机号后人工发送临时密码
3. 手机号已停用/更换：
   - 需用户提供实名认证信息
   - 核实身份后协助更换绑定手机号

### 原因3: 账户余额异常减少
**特征**: 用户反映余额莫名减少，无对应消费记录
**处理方案**:
1. 查询余额变动流水，找出扣款记录
2. 常见原因：
   - 自动扣除欠费订单
   - 违规罚款自动扣除
   - 账户被盗用消费
3. 如确认为欠费/罚款扣除：向用户说明明细
4. 如疑似账户被盗：
   - 立即冻结账户
   - 引导用户修改密码
   - 核实异常消费，退还被盗消费金额
   - 建议用户报警

### 原因4: 优惠券/骑行卡消失
**特征**: 用户反映账户内优惠券不见了
**处理方案**:
1. 查询优惠券使用/过期记录
2. 如已过期：向用户说明过期时间，酌情补偿 1 张同等优惠券
3. 如被异常使用（账户被盗）：按账户被盗流程处理
4. 如系统错误导致丢失：补发对应优惠券

### 原因5: 实名认证失败
**特征**: 用户反映实名认证一直不通过
**处理方案**:
1. 确认用户提交的证件类型（身份证/护照/港澳台证件）
2. 常见失败原因：
   - 证件照片模糊：引导用户重新拍摄
   - 证件信息与姓名不符：核实用户信息
   - 证件已过期：引导用户更新证件
   - 系统识别问题：人工审核通道处理

## 账户安全建议
向用户提供以下安全建议：
1. 不要将账户借给他人使用
2. 发现异常登录立即修改密码
3. 开启登录通知功能
4. 不要在公共设备上保存登录状态

## 补偿标准
- 因系统错误导致账户被误封：解封 + 补偿 **1张30分钟骑行券**
- 因账户被盗导致损失：退还被盗金额 + 补偿 **5元优惠券**
- 因系统问题导致优惠券丢失：补发等额优惠券

## 验证步骤
1. 确认账户状态已恢复正常
2. 引导用户测试登录和开锁
3. 确认补偿已到账
