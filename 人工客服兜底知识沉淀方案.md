# 人工客服兜底并自动沉淀知识库 - 改造方案

> 版本: v1.0 | 日期: 2026-03-28

---

## 一、功能概述

### 1.1 核心需求

当智能客服系统的知识库无法处理用户需求时，需要转人工处理。在人工处理完成后，系统自动根据以下信息生成知识文档并沉淀到知识库：

- 用户与人工客服的对话记录
- 该订单/工单的相关信息
- 问题类型和解决方案

### 1.2 替换的功能

**删除功能**：投诉处理分析报告生成（`ServiceAgentService` 的多 Agent 深度分析流程）

**新增功能**：人工客服对话记录 → 知识文档生成 → 自动入库

---

## 二、业务流程设计

### 2.1 完整业务流程

```
用户提问
    ↓
智能客服尝试回答（基于现有知识库）
    ↓
判断：是否需要人工介入？
    ├─ 否 → 继续智能客服对话
    └─ 是 → 转人工客服
            ↓
        人工客服处理
            ↓
        人工标记"问题已解决"
            ↓
        【触发知识沉淀】
            ↓
        系统调用 LLM 生成知识文档
            ↓
        自动上传到知识库（向量化存储）
            ↓
        完成
```

### 2.2 触发时机

**方式一（推荐）**：人工客服在前端点击"完成并沉淀知识"按钮

- 优点：简单直接，人工可控
- 适用场景：人工客服处理完问题后主动触发

**方式二（可选）**：系统自动检测对话结束

- 优点：全自动化
- 缺点：需要判断对话是否真正结束，可能误触发
- 本方案采用方式一

## 三、技术架构设计

### 3.1 核心组件

| 组件 | 职责 | 实现方式 |
|------|------|----------|
| `KnowledgeDepositService` | 知识沉淀服务 | 新增 Service |
| `POST /api/deposit_knowledge` | 知识沉淀接口 | 新增 Controller 方法 |
| 前端"完成并沉淀"按钮 | 触发入口 | 修改 `index.html` + `app.js` |
| LLM 知识文档生成 | 根据对话生成文档 | 使用 DashScope API |
| 文档自动上传 | 向量化并存储 | 复用 `FileUploadController` |

### 3.2 数据流

```
前端提交
  ↓
{
  "conversationHistory": [...],  // 用户与人工客服的对话记录
  "orderInfo": {...},            // 订单/工单信息（可选）
  "problemType": "开锁失败"      // 问题类型（可选）
}
  ↓
KnowledgeDepositService
  ↓
调用 LLM 生成知识文档（Markdown 格式）
  ↓
保存为临时文件
  ↓
调用 FileUploadController 上传
  ↓
向量化并存储到 Milvus
  ↓
返回成功
```

## 四、详细实现方案

### 4.1 后端改造

#### 4.1.1 新增 `KnowledgeDepositService.java`

**文件路径**：`src/main/java/org/example/service/KnowledgeDepositService.java`

**核心方法**：

```java
/**
 * 根据对话记录生成知识文档
 * @param conversationHistory 对话记录
 * @param orderInfo 订单信息（可选）
 * @param problemType 问题类型（可选）
 * @return 生成的 Markdown 文档内容
 */
public String generateKnowledgeDocument(
    List<Map<String, String>> conversationHistory,
    Map<String, Object> orderInfo,
    String problemType
)
```

**实现逻辑**：

1. 构建 Prompt，包含：
   - 对话记录
   - 订单信息
   - 问题类型
   - 要求 LLM 生成标准化的知识文档

2. 调用 DashScope API 生成文档

3. 返回 Markdown 格式的文档内容

**Prompt 模板**：

```
你是一个知识库文档生成专家。请根据以下信息生成一份标准化的客服知识库文档。

## 对话记录
[对话记录]

## 订单信息
[订单信息]

## 问题类型
[问题类型]

请生成一份 Markdown 格式的知识文档，包含以下部分：

# [问题标题]

## 问题描述
[用户遇到的问题]

## 适用场景
[什么情况下会遇到这个问题]

## 解决方案
[详细的解决步骤]

## 注意事项
[需要注意的事项]

## 相关政策
[相关的赔偿/补偿政策]

要求：
1. 标题简洁明了，不超过20字
2. 内容基于对话记录，不要编造
3. 解决方案要具体可操作
4. 语言专业、友好
```

#### 4.1.2 新增 Controller 接口

**文件**：`ChatController.java`

**新增方法**：

```java
/**
 * 人工客服知识沉淀接口
 * 根据对话记录生成知识文档并自动入库
 */
@PostMapping("/deposit_knowledge")
public ResponseEntity<ApiResponse<DepositResponse>> depositKnowledge(
    @RequestBody DepositRequest request
)
```

**请求参数**：

```java
public static class DepositRequest {
    private List<Map<String, String>> conversationHistory;  // 对话记录
    private Map<String, Object> orderInfo;                  // 订单信息（可选）
    private String problemType;                             // 问题类型（可选）
}
```

**响应参数**：

```java
public static class DepositResponse {
    private boolean success;
    private String documentContent;  // 生成的文档内容
    private String fileName;         // 保存的文件名
    private String message;
}
```

**实现逻辑**：

1. 调用 `KnowledgeDepositService.generateKnowledgeDocument()` 生成文档
2. 保存为临时 Markdown 文件（文件名：`knowledge_[timestamp].md`）
3. 调用 `FileUploadController` 的上传逻辑，将文档向量化并存储
4. 返回成功响应

**修改文件**：`ChatController.java`

**操作**：

**修改文件**：`ServiceAgentService.java`

**文件**：`app.js`

**新增功能**：

```javascript
// 1. 绑定"完成并沉淀知识"按钮点击事件
document.getElementById('depositKnowledgeBtn').addEventListener('click', () => {
    // 检查是否有对话记录
    if (currentSessionHistory.length === 0) {
        alert('当前没有对话记录，无法沉淀知识');
        return;
    }

    // 显示确认对话框
    document.getElementById('depositModal').style.display = 'flex';
});

// 2. 确认沉淀
document.getElementById('depositConfirmBtn').addEventListener('click', async () => {
    const problemType = document.getElementById('problemTypeSelect').value;
    const orderId = document.getElementById('orderIdInput').value;

    if (!problemType) {
        alert('请选择问题类型');
        return;
    }

    // 隐藏对话框
    document.getElementById('depositModal').style.display = 'none';

    // 显示加载状态
    showLoadingMessage('正在生成知识文档并沉淀到知识库...');

    try {
        // 调用后端接口
        const response = await fetch('/api/deposit_knowledge', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
                conversationHistory: currentSessionHistory,
                orderInfo: orderId ? { orderId } : null,
                problemType: problemType
            })
        });

        const result = await response.json();

        if (result.code === 200 && result.data.success) {
            // 成功提示
            showSuccessMessage('知识已成功沉淀到知识库 ✓');

            // 可选：显示生成的文档预览
            // showDocumentPreview(result.data.documentContent);
        } else {
            showErrorMessage('知识沉淀失败：' + result.data.message);
        }
    } catch (error) {
        showErrorMessage('知识沉淀失败：' + error.message);
    }
});

// 3. 取消沉淀
document.getElementById('depositCancelBtn').addEventListener('click', () => {
    document.getElementById('depositModal').style.display = 'none';
});
```

## 六、核心优势

### 6.1 简单性

- **触发方式简单**：一个按钮点击即可
- **无需复杂判断**：人工决定何时沉淀
- **实现简单**：复用现有的文档上传和向量化逻辑

### 6.2 实用性

- **真实场景**：人工客服处理完问题后立即沉淀
- **质量可控**：人工可以选择是否沉淀（避免无效对话入库）
- **信息完整**：包含对话记录、订单信息、问题类型

### 6.3 可扩展性

- **未来可增强**：
  - 自动检测对话结束并提示沉淀
  - 支持编辑生成的文档后再入库
  - 支持批量沉淀多个对话
  - 支持知识文档审核流程

## 七、与原功能的对比

| 维度 | 原投诉分析功能 | 新知识沉淀功能 |
|------|---------------|---------------|
| **目标** | 深度分析投诉，生成详细报告 | 快速沉淀知识，扩充知识库 |
| **触发** | 主动点击"投诉分析"按钮 | 人工处理完成后点击 |
| **输入** | 无需输入（自动查询工单） | 当前对话记录 + 订单信息 |
| **处理** | 多 Agent 协作，深度挖掘 | 单次 LLM 调用，快速生成 |
| **输出** | 长篇分析报告（仅展示） | 标准化知识文档（入库） |
| **价值** | 帮助理解复杂投诉 | 持续扩充知识库，提升智能客服能力 |
| **复杂度** | 高（多 Agent，多工具调用） | 低（单次 LLM 调用 + 文档上传） |

## 九、注意事项

### 9.1 数据隐私

- 对话记录可能包含用户隐私信息（手机号、订单号等）
- 建议在生成文档时脱敏处理
- 或在 Prompt 中要求 LLM 自动脱敏

### 9.2 文档质量

- LLM 生成的文档质量取决于对话质量
- 建议人工客服在对话时保持专业、结构化
- 可选：增加人工审核环节（生成后预览，允许编辑）

### 9.3 知识库管理

- 定期检查沉淀的知识文档质量
- 删除重复或低质量的文档
- 可增加文档分类和标签功能
